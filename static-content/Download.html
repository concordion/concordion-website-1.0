Changes in Concordion 1.4.6 — 23rd November 2014

  • The results of the run command are now cached, and the cached result returned if the test is run subsequently within the test run (#94). If you're using the run command, this:
    1. allows a specification to be referenced from multiple index pages using concordion:run, with the pass/fail/ignore result showed on each reference link.
    2. removes the need to only include the fixture for your main index specification in the test configuration of your build script. You can now include all fixtures in the build script, and each will only run once. This also ensures that any "orphaned" fixtures (that aren't referenced in your specifications) are run.
    3. allows you to invoke a test run of all fixtures in your IDE, with each only being run once.
    Note that the cache only works within a single JVM instance and caching will not work if you fork additional JVMs to run the fixtures.
  • The execute command now has special behavior when placed on a list element. (#92)
  • Added support for lower case commands to comply with the XHTML standard. (#88)
  • Added a listener for the set command. (#96)
  • The reference specs now use the run command. (#75)
  • The reference fixtures now use the JUnit 4 ConcordionRunner. (#74)
  • Gradle Wrapper added, so that users don't need to download Gradle manually, and ensure correct Gradle version is used (#93)
  • Fixed Gradle deprecation warning. (#80)
  • Added Travis CI build config
Thanks to Tim Wright, Jacek Ratzinger, John Hurst, Dan Katzel and Nigel Charman for their commits.

Concordion 1.4.3 — 26th January 2013

Upgraded dependencies

Concordion has been upgraded to the latest version of JUnit (JUnit 4.11). This change has caused a new dependency on Hamcrest.

If you're using Maven, Hamcrest is included in the transitive dependencies of JUnit, so should be added to your classpath automatically.

If you're installing or upgrading Concordion manually, make sure you add the file "hamcrest-core-1.3.jar" to your classpath, in addition to the other jar dependencies (all of which can be found in the Concordion zip file in the "lib" directory). Failure to do so may result in the error "java.lang.NoClassDefFoundError: org/hamcrest/SelfDescribing".

Other improvements
  • Added a MultiValueResult class to make it easier to return multiple results from a fixture method
  • Allow the assertEquals command to match "Yes/No" "True/False" "Y/N" with a Boolean returned from the system under test
  • Added support to concordion:run for test classes ending in Fixture (thanks to Ricardo da Silva Lima for the patch)
  • Modified CSS on verifyRows to show missing rows as struck-through
  • In Extensions API: Added support for Element to look for other elements by id (thanks to Alessandro Novarini for the patch)
Bug-fixes
  • Fixed: Error message shown for all @Unimplemented tests, not just those with an implementation
  • Fixed: Duplicate extension instances created due to inheritance of @Extension annotation (thanks to Bryan Young for the issue report and unit test)

Concordion 1.4.4 — 18th August 2013

  • Added a FailFast annotation to stop processing the current specification after an exception occurs.
  • Added withSpecificationLocator() to the extension API. This provides the ability to read and write to files using different file suffixes by configuring an extension. (Note, this includes an update to the Target API to include an additional method).
