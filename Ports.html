Supported Technologies

Concordion has been ported to the following languages:

  • Java
  • .NET
  • Python
  • Fantom
  • Ruby
  • Scala

Java

Concordion

This is the initial development of Concordion, written by David Peterson, an agile consultant based in London, UK.

.NET

Concordion.NET

This is a very faithful port of Concordion, written by Jeffrey Cameron, for the .NET framework. All commands, including the annotations (or attributes in .NET-speak) and the concordion:run command, introduced in 1.3.1, are implemented, and behave the same way as in the Java version, with only minor differences to make it more .NET friendly.

Python

PyConcordion

This is a Python adapter for the Java version. PyConcordion is developed by JC Plessis.

PyConcordion2

John Jiang is currently developing PyConcordion2. The work is still in progress but it's 100% python thus easier to install/use.

Fantom

Fancordion

Fancordion is an adaptation of Concordion for the Fantom JVM language. Conceptual differences include:

  • Specifications written in fandoc - a plain text, markdown like, syntax
  • Specifications written as class comments, alongside the code

See the API Documentation for more details.

Developed and maintained by Steve Eynon under the Alien-Factory pseudonym.

Ruby

ruby-concordion

Developed by Ben Goodspeed, Ariel Valentin and Chris Gardner.

Ariel Valentin has also written a simple getting started guide for using Ruby Concordion with Rails 2.0+.

This project is no longer active.

Scala

accordion-scala

Developed by Tom Malone.
