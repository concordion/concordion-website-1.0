Welcome to the Concordion community blog. The primary goal of this blog is to deliver interesting content for the software development community. If you have valuable ideas, tips, or experiences, then you are more than welcome to contribute. Sharing practical examples are important. After reading your blog post, your readers should not only have gained new insights and ideas, but also know how to actually implement these in practice. We also want you to share your expertise, things you have learned throughout your career and things you wish you had known earlier. The more practical your tips, techniques and ideas are, the better.

This community blog is based on Jekyll to allow blogging using simple static HTML web pages, with all of the content hosted on the version controll plattform of GitHub. The goal was to eliminate the complexity of other blogging platforms by creating a workflow that allows you to blog like a hacker.. To create a new blog entry you shold get started with the following steps:

  1. Create your own fork of the blog.
  2. Copy the blog template and rename it according to the topic you want to write about.
  3. Edit the header information and add your article content.
  4. Create a pull request to get your article published.

1. Create your own fork of the blog

A fork is a copy of the website repository containing all blog entries. Forking a repository allows you to freely experiment with changes without affecting the original project. You can accomplish this step directly in the browser. Got to the blog repository on github.com and click on fork in the top-right corner of the page. That's it! Now, you have your private copy of the original blog repository.

2. Rename the template with the title of your post

To write a new blog post, you have to create a new HTML file within the directory that contains all existing blog entries (_posts). To achive this step in a web browser, navigate to the _post directory of your fork of the blog that you have created in the previous step. Click on the plus sign on top of the page (concordion-website/static-content/_posts/+). Set the file name to include today's date and the title of your post. The naming convention requires posts to be named year-month-day-title.html.

The _posts directory also contains template for your article ( _yyyy-mm-dd--title-of-your-post.html). Open it in the browser and copy the content into the new blog post you have just created. Save your new post file with the copied content from the template file by pressing the propose-new-file button on the lower end of the page.

3. Write your blog content

Now you can edit the content of your post file. First you should edit the first lines of the file containing header information of your article. Please, provide a title summarizing the idea, tip, or experience you want to share. Additionally, provide your name next to the author tag and the URL of your own web page, if you want to backlink to it. This will be displayed at the top of your blog post, similar to the information you see in this blog example.

Please, describe your valuable tips, ideas, or experiances in between the <article> tags further down in the template text. You can either type your content using HTML syntax directly in the browser or clone the fork of your blog onto your computer and use one of the available html editors. WYSIWYG editors help you to write your content in a Word-like fashion.

<article>
  Please, add your article content here.
</article>

4. Publish your blog post

Your blog post is now written and looking good - you have just to publish it. This is done through issuing a pull request on github.com. Open the fork of the blog you have created in step 1 in your web browser. Click on the create-pull-request button on top of the screen. Add a short comment about the idea of your blog post and press the confirming create-pull-request submit button on the bottm of the page. After the pull request has been accepted, you can view your blog on the Concordion webpage.
